## 产品说明

### 产品定位

专为珠宝零售门店打造的**轻量级销售数据分析工具**。无需安装任何软件，打开网页上传 Excel 即可获得专业的销售分析报告。

**目标用户**：
- 珠宝店老板 / 店长
- 需要快速了解经营状况的零售从业者
- 希望用数据驱动决策的小微企业主

### 核心功能

| 功能 | 描述 | 状态 |
|------|------|------|
| 📤 Excel 上传 | 支持 .xls / .xlsx 格式，拖拽或点击上传 | ✅ 已完成 |
| 📊 自动分析 | 6 大分析维度，覆盖销售核心指标 | ✅ 已完成 |
| 📈 可视化报告 | Chart.js 图表 + 交互式表格 | ✅ 已完成 |
| 📜 历史记录 | 所有分析记录永久保存，随时回看 | ✅ 已完成 |
| 📥 报告下载 | 导出 HTML 格式报告，离线查看 | ✅ 已完成 |
| 📱 多端适配 | PC / 平板 / 手机全响应式 | ✅ 已完成 |
| 🤖 AI 分析 | 调用 Minimax 大模型进行智能分析 | 🔜 开发中 |

### 分析维度详解

#### 1️⃣ 营业额 & 利润统计

```
统计范围：最近 30 天（以文件中最新日期为基准向前推算）

输出指标：
├── 总营业额（业绩金额合计）
├── 总利润（利润金额合计）
└── 销售笔数（业绩金额 > 0 的交易数）
```

#### 2️⃣ 价格区间业绩排名

将所有销售按业绩金额分为 7 个价格区间，统计各区间的表现：

| 区间 | 范围 |
|------|------|
| 0-100 元 | 入门款 |
| 100-300 元 | 消费级 |
| 300-500 元 | 中档 |
| 500-1000 元 | 主力价位 |
| 1000-2000 元 | 高端 |
| 2000-5000 元 | 奢华 |
| 5000 元以上 | 顶级 |

每个区间输出：交易笔数、总金额、平均客单价、明细列表（可点击展开）

#### 3️⃣ 品类业绩对比

按「小类」字段分组统计：

```
输出格式：
├── 品类名称（如：金包银、黄金、银饰等）
├── 该品类总业绩
└── 该品类交易笔数

展示形式：柱状图（Chart.js），按业绩降序排列
横轴：品类名称
纵轴：业绩金额
```

#### 4️⃣ 热销商品 TOP 8

按商品名称汇总，取业绩最高的前 8 名：

```
输出格式：
├── 商品名称
├── 总业绩
└── 销售件数

展示形式：横向柱状图（Chart.js）
```

#### 5️⃣ 旧料回收明细 & 盈亏分析 ⭐ 核心特色

这是本产品的**差异化功能**，针对珠宝行业特有的旧料回收业务：

```
筛选条件：销售类型 == '回收'

每笔回收记录计算：
├── 金属类型判断（金/银）：根据商品名称关键词匹配
│   ├── 金：包含 '金'、'AU'、'au'、'素圈'
│   └── 银：其他
├── 回收重量：取「金重」/「银重」字段的绝对值
├── 支付金额：取「业绩金额」的绝对值
├── 当前市值 = 回收重量 × 当前单价
│   ├── 金价默认 ¥930/g
│   └── 银价默认 ¥17/g
└── 盈亏 = 当前市值 - 支付金额

汇总输出：
├── 回收总支出（实际支付金额合计）
├── 当前持有市值（金重×金价 + 银重×银价）
├── 整体盈亏
├── 金料明细：总重量、支出、当前价值、笔数
└── 银料明细：总重量、支出、当前价值、笔数
```

#### 6️⃣ 明细钻取（Drill-down）

所有聚合数据支持点击展开查看原始明细：
- 价格区间 → 该区间的每一笔交易
- 品类 → 该品类的每一个商品
- 回收 → 每一笔回收的详细数据

##### POST `/api/upload` — 上传并分析文件

**请求**：
```
Content-Type: multipart/form-data

参数：
├── file (required)  Excel 文件 (.xls/.xlsx)
└── 最大大小: 16MB
```

**成功响应 (200)**：
```json
{
  "id": "70cef442-ee5e-4e42-ae05-27a83595feb2",
  "file_name": "销售明细查询_2026-06-18.xls",
  "total_revenue": 19801.50,
  "total_profit": 5642.30,
  "report_url": "/api/report/70cef442-ee5e-4e42-ae05-27a83595feb2"
}
```

**错误响应**：
```json
// 400 - 文件格式不支持
{ "detail": "只支持 .xls 或 .xlsx 文件" }

// 500 - 分析失败
{ "detail": "分析失败: 列名不匹配" }
```

### 数据模型

#### AnalysisRecord（分析记录表）

```sql
CREATE TABLE analysis_records (
    id          VARCHAR(36) PRIMARY KEY,     -- UUID
    file_name   VARCHAR(255) NOT NULL,       -- 原始文件名
    upload_time DATETIME NOT NULL,           -- 上传时间
    total_revenue FLOAT DEFAULT 0,           -- 营业总额
    total_profit  FLOAT DEFAULT 0,           -- 总利润
    report_html   TEXT,                      -- HTML报告内容
);
```

#### Excel 数据格式要求

上传的 Excel 必须包含以下列（列名需完全匹配）：

| 列名 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 销售时间 | datetime | ✅ | 交易发生时间 |
| 销售类型 | string | ❌ | 如"销售"、"回收"等 |
| 小类 | string | ✅ | 商品分类（如"金包银"、"黄金"） |
| 商品名称 | string | ✅ | 具体商品名称 |
| 销售金额 | float | ❌ | 原始销售金额 |
| 业绩金额 | float | ✅ | 实际业绩核算金额 |
| 利润金额 | float | ✅ | 单笔利润 |
| 金重 | float | ❌ | 回收金料重量（克） |
| 银重 | float | ❌ | 回收银料重量（克） |
| 件重 | float | ❌ | 商品总重量 |

**Web Service 配置**：
| 配置项 | 值 |

1. 在 Render 创建 Free PostgreSQL 数据库

**环境变量**：
| 变量名 | 说明 | 示例 |

| DATABASE_URL | PostgreSQL 连接串 | `postgresql://user:pass@host:5432/dbname` |

## Analysis Prompt 模板 ⭐

> 以下是当前各分析维度的 Prompt 描述，用于后续集成 Minimax API 时作为分析指令。

### Prompt 1: 营业额 & 利润概览

```
你是一个专业的珠宝零售数据分析师。请对用户上传的销售明细Excel数据进行以下分析：

## 任务1：营业额与利润统计

### 输入
- Excel文件中的销售明细数据

### 统计规则
1. 时间范围：以文件中最新的销售日期为基准，向前推算30天
2. 营业额计算：统计范围内所有「业绩金额」字段的合计
3. 利润计算：统计范围内所有「利润金额」字段的合计
4. 销售笔数：业绩金额 > 0 的交易记录数量

### 输出格式
请返回JSON格式的结果：
{
  "total_revenue": 营业总额,
  "total_profit": 总利润,
  "sales_count": 销售笔数,
  "time_range": {
    "start": 统计起始日期,
    "end": 统计截止日期
  }
}
```

### Prompt 2: 价格区间分析

```
## 任务2：价格区间业绩排名

### 分组规则
将所有销售记录（业绩金额 > 0）按以下价格区间分组：

| 区间标签 | 金额范围(元) |
|---------|-------------|
| 0-100元 | [0, 100] |
| 100-300元 | (100, 300] |
| 300-500元 | (300, 500] |
| 500-1000元 | (500, 1000] |
| 1000-2000元 | (1000, 2000] |
| 2000-5000元 | (2000, 5000] |
| 5000元以上 | (5000, +∞) |

### 每个区间输出
- 交易笔数
- 业绩金额合计
- 平均客单价
- 明细列表：每笔交易的[日期、商品名称、小类、金额]

### 返回JSON
{
  "range_stats": [
    { "range": "区间名", "count": 笔数, "total": 合计, "avg": 平均 }
  ],
  "range_details": {
    "区间名": { "transactions": [...], "total_amount": ..., "count": ... }
  }
}
```

### Prompt 3: 品类对比分析

```
## 任务3：品类业绩对比

### 规则
- 按「小类」字段对所有销售记录（业绩金额 > 0）进行分组
- 计算每个品类的总业绩和交易笔数
- 按总业绩降序排列

### 输出

  ]

### 图表要求

- X轴：品类名称
- Y轴：业绩金额
- 颜色建议：渐变红色系（#e74c3c → #c0392b）
```

### Prompt 4: 热销商品排行

```
## 任务4：热销商品 TOP 8

### 规则
- 按「商品名称」字段对所有销售记录（业绩金额 > 0）进行分组
- 计算每个商品的总业绩和销售件数
- 按总业绩降序排列，取前8名

### 输出
{
  "prod_stats": [
    { "name": "商品名称", "count": 件数, "total": 业绩合计 }

### 图表要求

- Y轴：商品名称
- X轴：业绩金额
- 仅显示TOP8
```

### Prompt 5: 回收业务分析（核心特色）

```
## 任务5：旧料回收明细与盈亏分析 ⭐珠宝行业专属

### 筛选条件
- 销售类型字段值为"回收"的所有记录

### 参数（可由用户自定义，提供默认值）
- 金价：¥930/克（默认）
- 银价：¥17/克（默认）

### 每条记录处理逻辑
1. **金属类型判断**：
   - 如果商品名称包含 ['金','AU','au','素圈'] → 黄金
   - 否则 → 白银

2. **重量提取**：
   - 黄金：取「金重」字段的绝对值
   - 白银：取「银重」字段的绝对值

3. **支付金额**：取「业绩金额」字段的绝对值

4. **当前市值计算**：
   - 当前市值 = 重量 × 对应当前金属单价

5. **单笔盈亏**：
   - 盈亏 = 当前市值 - 支付金额
   - 正数为盈利，负数为亏损

### 汇总输出
{
  "recycle_items": [  // 每笔回收明细
    {
      "date": "MM-DD HH:mm",
      "name": "商品名称",
      "metal": "gold/silver",
      "weight": 重量(克),
      "paid": 支付金额,
      "value": 当前市值,
      "pnl": 盈亏,
      "price": 使用的单价
    }
  ],
  // 汇总统计
  "recycle_total_paid": 总支出,
  "recycle_total_value": 当前总市值,
  "recycle_total_pnl": 整体盈亏,
  // 金料汇总
  "recycle_gold_g": 金重合计(克),
  "recycle_gold_paid": 金料支出,
  "recycle_gold_value": 金料当前价值,
  "recycle_gold_count": 金料笔数,
  // 银料汇总
  "recycle_silver_g": 银重合计(克),
  "recycle_silver_paid": 银料支出,
  "recycle_silver_value": 银料当前价值,
  "recycle_silver_count": 银料笔数,
  // 使用的单价
  "gold_price": 使用的金价,
  "silver_price": 使用的银价
}
```

### Prompt 6: 完整综合分析（Minimax API 主 Prompt）
